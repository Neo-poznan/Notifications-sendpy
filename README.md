# Проект Notifications-sendpy

* Позволяет рассылать сообщения по электронной почте большому количеству людей.
* В качестве данных отправителя использует электронную почту пользователя
* Проверка данных пользователя на валидность в "боевых" условиях и гарантия доставки сообщений "Минимум 1 раз"
* Контакты получателей загружаются в виде CSV файла.
* HTML текстовый редактор для верстки сообщения
* Проект легко масштабируется: может быть задействовано большое количество воркеров для огромных рассылок.

## Простая и понятная инструкция по использованию сервиса находится на главной странице + инструкция в картинках по настройке своей электронной почты

## Как работает?
Когда пользователь авторизовался, загрузил данные получателей, пароль приложений и верстку сообщения и нажал кнопку "отправить сообщение" и все данные введены верно, то список контактов делится на несколько частей равное количеству партиций и в несколько потоков (для ускорения) отправляются данные для формирования сообщений. Каждый поток формирует сообщения для своей партиции. Для каждого адреса электронной почты формируется одно сообщение, которое содержит в себе контакт получателя, сообщение, его заголовок, электронную почту пользователя и пароль приложений. Это сообщение попадает в кафку. Из кафки его принимает воркер, который парсит сообщение, создает SMTP сервер и отправляет сообщение на почту получателя. 

## Как запустить у себя?
1. Создать директорию
2. Создать в ней виртуальное окружение `python3.12 -m venv env` а затем активировать его `source env/bin/activate`
3. Создать внутри первой директории еще одну
4. Клонировать в нее репозиторий
5. Настроить settings.py прописать в нем настройки kafka и базы данных
6. Создать у себя переменную окружения DJANGO_KEY и поместить в нее свой ключ django
7. Перейти в терминале в директорию куда вы клонировали код
8. Установить зависимости: `pip install -r requirements.txt`
9. Провести миграции `python3.12 manage.py makemigrations` а потом `python3.12 manage.py migrate`
10. `python3.12 manage.py runserver`

## Как использовать api
Чтобы отправить рассылку из api нужно отправить post запрос на адрес `http(s)://ваш-домен/api/send-message` в теле запроса должен быть json с парами ключ-значение: ключ message со значение в виде строки с версткой сообщения, ключ message_header с значением строкой с заголовком сообщения, ключ recipient_email_list со значением в виде массива со строковыми значениями электронной почты получателей, ключ sender_email со значением строки с вашей электронной почты и ключ app_password со значением пароля приложений с проставленными вручную пробелами(иначе может не сработать)
